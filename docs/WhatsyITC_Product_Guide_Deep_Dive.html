Product concept

WhatsyITC turns bulk WhatsApp work into a guided operating system.

Many teams already use WhatsApp for follow-ups. The problem is not sending one message. The problem is managing hundreds of replies, missed leads, errors, and next steps.

InputBatches

Spreadsheet rows with retailers, phone numbers, invoice or offer details.

ChannelWhatsApp

Template sends, replies, statuses, and unified chat history.

AutomationAI Agent

Context-aware replies grounded in knowledge and recent messages.

FocusReview

Important phone numbers rise to the human review queue.

The core business value

  • Reduce manual follow-up effort after a batch send.
  • Keep every phone conversation visible in one place.
  • Let AI answer routine questions using business knowledge.
  • Escalate only the conversations that need a person.
  • Give managers a batch-wise view of what is happening.

The user experience principle

The system should not feel like a hidden automation box. It should feel like a control room where users can see the current state, understand why something happened, and change the next action.

Visible status Editable plans Warnings Human handoff

What happens when a customer replies

  • Webhook receives the WhatsApp message.
  • The phone is mapped to the correct admin and batch when possible.
  • The inbound message is saved to the thread.
  • AI checks batch ownership, handoff, agent status, and recent history.
  • AI replies or the conversation is surfaced for human review.

What the operator sees

  • AI Conversations show AI-managed threads.
  • Human Review shows only numbers that need action.
  • AI CRM shows batch summaries and lead priority.

Batch operations

Uploads, approval, templates, resend, and naming.

The batch workflow is the starting point for most customers. It keeps mass messaging controlled and auditable.

Batch preparation

  • Upload CSV or XLSX files with phone numbers and business details.
  • Validate rows before sending.
  • Keep original file name for audit.
  • Set a user-friendly batch display name.
  • Review duplicates and invalid numbers before enabling AI.

Sending and resending

  • Select the right WhatsApp template.
  • Use consent dialogs before important sends.
  • Track statuses like queued, sent, delivered, failed, and received.
  • Resend to a batch when a fresh campaign message is needed.
  • Keep errors visible so users can correct credentials or templates.

Duplicate phone handling

If the same phone appears in multiple batches, the user should understand ownership before enabling AI. The system should warn that another batch may already have an AI agent for that phone, then let the user skip that phone or override ownership with the current batch agent.

Warn first Skip duplicate Override ownership Clear batch agent

Knowledge system

How long context becomes useful AI memory.

Large documents should not be passed fully into every reply. They should be split, embedded, searched, and only the most relevant parts should be sent to the model.

1
Paste textProduct docs, policy notes, FAQs, price rules.
2
ChunkSplit into useful sections without losing meaning.
3
EmbedCreate vectors for semantic search.
4
RetrieveFind best chunks for the user question.
5
AnswerReply using chat history plus retrieved facts.

Why this is better than pasting everything

  • Lower token usage per message.
  • Less irrelevant information in the prompt.
  • Better answers when documents are long.
  • Knowledge can be updated without changing code.
  • Embeddings make similar wording easier to find.

Best content to add

  • Product catalog and categories.
  • Pricing and discount rules.
  • Delivery timelines and service areas.
  • Complaint handling policies.
  • Common questions and approved answers.
Practical rule: add knowledge in clean sections with descriptive titles. The AI performs best when each chunk answers one clear topic.

AI follow-ups

Batch-specific automation with human controls.

AI follow-ups are not global magic. They are controlled per batch so the user knows which agent owns which phone numbers.

Batch control room

  • Shows current AI state for the batch.
  • Shows the resolved agent and whether AI is enabled.
  • Shows tracked recipients and individual control rooms.
  • Allows editing or generating the next message.
  • Allows manual send-next confirmation before action.

Recipient control room

  • Displays the selected phone's latest activity.
  • Shows timeline in a dialog to avoid page clutter.
  • Shows message plan and AI reasoning signals.
  • Lets the user open full conversations.
  • Keeps controls near the selected recipient.

Next-message generation

When the user asks AI to generate the next follow-up, the system should use the last 10 to 20 relevant messages, batch objective, agent prompt, and knowledge. The generated message can be edited before being saved or sent.

Human review

Urgency queue for phone numbers that need judgment.

The human review queue keeps operators away from noise. It should show distinct phone numbers, why they matter, and what to do next.

Signals that can create urgency

  • Customer asked for a person.
  • Message delivery failed.
  • Complaint or negative sentiment.
  • Price question or buying intent.
  • Product confusion or unclear requirement.
  • Meeting or callback request.

What the queue should show

  • Phone number and retailer name.
  • Severity and reason label.
  • Last message preview.
  • Suggested action.
  • AI help for summary and draft reply.
  • Resolve or snooze controls.

Efficient AI usage

The best approach is to update review signals when the system is already analyzing a conversation for AI reply. That means one LLM call can both prepare a response and update review intelligence. Rule-based signals can still run cheaply for every message.

Rule first AI when already replying Cache advice Avoid duplicate phones

Operating playbooks

How teams should use it day to day.

A product like this succeeds when users have simple habits. These playbooks keep work repeatable.

Daily operator routine

  • Open Human Review first.
  • Handle critical and high severity numbers.
  • Use AI help for concise drafts.
  • Open Chats for full context when needed.
  • Resolve or snooze after action.

Manager routine

  • Open AI CRM.
  • Review action required by batch.
  • Check buyer replies and cadence.
  • Open batch detail for problem batches.
  • Update plan or agent if needed.

Admin routine

  • Check WhatsApp credentials.
  • Refresh templates when needed.
  • Improve knowledge base.
  • Test agent in playground.
  • Watch logs for send or LLM errors.

Troubleshooting mindset

If a reply does not send, check in this order: WhatsApp token, phone number ID, template validity, batch AI enabled state, conversation handoff status, agent enabled state, LLM provider error, and whether the phone belongs to the expected batch.

Implementation best practices

How to keep the experience excellent.

AI products fail when users cannot trust the output or cannot see why something happened. These practices protect trust.

For accurate AI replies

  • Keep knowledge chunks clean and topic-specific.
  • Test common questions in the agent playground.
  • Use product facts instead of generic marketing text.
  • Never ask for phone number when the thread already has it.
  • Keep replies short, warm, and WhatsApp-native.

For operator trust

  • Show send errors directly in the UI.
  • Use confirmation dialogs before sends and overrides.
  • Keep duplicate-phone warnings clear.
  • Make handoff and AI enabled states visible.
  • Do not hide messages across different inboxes.

Good first rollout

  • Start with one small batch.
  • Use one strong agent and one clear knowledge set.
  • Test with two verified WhatsApp numbers.
  • Simulate common buyer questions.
  • Only then scale batch size.

Things to avoid

  • Do not enable AI on messy duplicate batches without review.
  • Do not rely on AI without knowledge for pricing or policy answers.
  • Do not ignore failed sends or 401 credential errors.
  • Do not let stale handoff states block active batch AI.
  • Do not overload operators with every low-priority message.
