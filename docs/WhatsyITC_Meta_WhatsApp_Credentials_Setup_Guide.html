Step 2

Find the Phone Number ID and access token.

These two values are the minimum required for WhatsyITC to send WhatsApp messages through Meta Cloud API.

Phone Number ID

This identifies the WhatsApp sender number. It is different from the visible phone number.

  • Found on WhatsApp API Setup page.
  • Usually a long numeric value.
  • Used by the backend send endpoint.
  • Used to route inbound webhook payloads to the right admin.

Access token

This authorizes the backend to call Meta APIs. Temporary tokens are useful for testing; production should use a stable system user token.

  • Temporary token can expire quickly.
  • Permanent token should be rotated safely.
  • Save it only in WhatsyITC Settings.
  • If WhatsApp returns 401, re-check this token first.

Step 3

Add credentials inside WhatsyITC.

WhatsyITC stores WhatsApp credentials per admin user. That allows different admins to connect different WhatsApp Business accounts.

Verify TokenCreate a private string and reuse it in Meta webhook.
WABA IDOptional, paste if available.

After saving

  • Click Test connection.
  • Expect a verified or success status.
  • If failed, check token and phone number ID.
  • Keep the same verify token for webhook setup.

Security note

  • Do not store tokens in PDFs or screenshots.
  • Do not commit tokens into Git.
  • Rotate tokens when someone leaves the team.
  • Remove old credentials instead of leaving them active.

Step 4

Create the public webhook callback URL.

Meta must reach your backend over public HTTPS. Localhost cannot be used directly in Meta webhook configuration.

Local development

Use a tunneling service such as ngrok to expose backend port 8082.

ngrok http 8082

Copy the generated HTTPS URL and add the webhook path.

Production

Use your real public backend domain with HTTPS.

https://api.yourdomain.com/webhook/whatsapp

This URL must route to the Go backend webhook handler.

WhatsyITC webhook path

Use this exact path after your public backend domain or tunnel URL:

/webhook/whatsapp

Examples:

https://abc-123.ngrok-free.app/webhook/whatsapp
https://your-backend-domain.com/webhook/whatsapp
If Meta cannot verify the URL, confirm the backend is running, the URL is public HTTPS, the path is correct, and the verify token matches exactly.

Step 6

Add test phone numbers.

In development mode, Meta only allows messages to phone numbers you add and verify in the test list.

Where in Meta

  • Open WhatsApp API Setup.
  • Find Send messages with the API.
  • Click Manage phone number list near the To field.
  • Add your test phone number.
  • Verify the phone using OTP.

How to test

  • Send a test message from Meta first.
  • Confirm it arrives on WhatsApp.
  • Reply from that phone.
  • Check backend logs for webhook hit.
  • Open Chats and AI Conversations in WhatsyITC.
For real customer numbers outside the test list, complete production WhatsApp setup and use approved message templates.

Success signs

  • Settings shows verified or success.
  • Webhook verify succeeds in Meta.
  • Backend logs show webhook hit.
  • Incoming buyer reply appears in Chats.
  • AI Conversations can show the same phone thread.

Failure signs

  • Meta returns 401 authentication error.
  • Webhook verify fails or returns forbidden.
  • No backend log when phone replies.
  • Template name not found or not active.
  • Message appears under wrong admin or wrong batch.

Troubleshooting

Common problems and fixes.

Most Meta setup issues come from mismatched tokens, expired access, wrong callback URL, or template mismatch.

Problem
Likely cause
Fix
401 Authentication Error
Access token expired, revoked, or saved for wrong sender.
Generate/rotate token in Meta and re-save in Settings.
Webhook verify fails
Verify token mismatch or backend URL not reachable.
Paste exact same verify token and confirm HTTPS URL reaches backend.
No inbound messages
Webhook fields not subscribed or wrong phone number ID routing.
Subscribe messages/statuses and confirm phone number ID in Settings.
Only test numbers work
App is in development/test mode.
Add test numbers or complete production business setup.
Template send fails
Template not approved, wrong name, wrong language, or wrong variables.
Match Meta template exactly with WhatsyITC template record.
Wrong admin receives webhook
Multiple admins share callback but phone number ID not claimed correctly.
Save unique phone number ID for each admin credential.
Do not fix token issues by pasting tokens into code or `.env` if the product is using Settings-based per-user credentials. Save the token in the admin Settings screen.
