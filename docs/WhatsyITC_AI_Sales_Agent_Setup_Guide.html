Step 1

Set up WhatsApp credentials.

Credentials connect the system to Meta WhatsApp Cloud API. Without this, the app may show conversations but cannot reliably send or receive real WhatsApp messages.

What the user needs

  • Meta WhatsApp phone number ID.
  • WhatsApp access token.
  • Webhook verify token.
  • Correct API version.
  • Webhook URL configured in Meta.

What this enables

  • Send approved WhatsApp template messages.
  • Receive incoming buyer replies.
  • Track statuses like sent, delivered, read, failed.
  • Attach inbound messages to the correct admin and phone.
  • Let AI respond after receiving messages.

Step 2

Create and configure the AI sales agent.

The agent is the personality and decision layer. It tells the AI how to speak, what role it has, and how it should behave in WhatsApp conversations.

Agent identity

  • Name shown internally.
  • Friendly sales tone.
  • Short WhatsApp style replies.
  • Human touch, not robotic language.

Model settings

  • Primary chat model.
  • Fallback behavior if configured.
  • Embedding model for knowledge search.
  • Environment-controlled provider keys.

System prompt

  • Explain business role.
  • Use known phone number; do not ask again.
  • Answer using knowledge.
  • Ask for human help when unsure.

Good prompt behavior

The agent should sound like a helpful sales person on WhatsApp: concise, warm, context-aware, and never over-explaining. It can use a light emoji when natural, but the message should still feel professional.

Add knowledge the agent can use.

Knowledge is where the user teaches the AI about products, prices, policies, delivery, FAQs, and common sales answers.

Best knowledge to add

  • Product categories and catalog details.
  • Pricing, discounts, and minimum order rules.
  • Delivery areas and timelines.
  • Return, refund, or complaint policies.
  • Common objections and approved answers.

How to write good knowledge

  • Use clear headings.
  • Keep one topic per section.
  • Use direct facts, not vague marketing.
  • Update old prices immediately.
  • Test with real customer questions.
For long documents, the system should split text into chunks and use embeddings so the AI retrieves only the most relevant facts. This keeps answers faster, cheaper, and more accurate.

Step 5

Enable AI follow-ups for a batch.

AI follow-ups are batch-specific. This keeps ownership clear when the same phone appears in different uploads.

Before enabling AI

  • Batch has valid WhatsApp numbers.
  • Initial template send is complete or planned.
  • Agent has been tested.
  • Knowledge is available.
  • Duplicate phone warnings are reviewed.

Enable dialog choices

  • Behavior mode: default, custom, or intelligent.
  • Timeline: send every few days.
  • Maximum number of follow-up messages.
  • Agent selection for the batch.
  • Skip or override duplicate phone ownership.

What happens after enabling

The system tracks recipients in that batch, links each phone to the selected agent, watches inbound replies, and generates follow-up messages based on cadence, chat history, knowledge, and the batch objective.

Conversations

Understand what the buyer and agent said.

Users should be able to open a phone thread and see both sides of the conversation: buyer messages, AI messages, human messages, templates, and errors.

Chats

The raw WhatsApp conversation view. Use this to inspect the full phone timeline regardless of whether AI is active.

  • Buyer inbound messages.
  • Outbound templates.
  • Manual replies.
  • Status and failures.

AI Conversations

The AI-managed conversation inbox. Use this to monitor threads the agent is handling or has handled.

  • AI replies.
  • User replies.
  • Handoff status.
  • Take over and hand back controls.
Both views should show the same phone history. The difference is purpose: Chats are general message history; AI Conversations are for managing the AI-assisted thread.

Human review

Handle important numbers first.

Human Review is the priority queue. It tells the user which phone numbers need attention, why they matter, and what action to take.

Why a number appears

  • Buyer asked for a human.
  • Message failed to send.
  • Buyer asked a price or buying question.
  • Buyer showed meeting intent.
  • AI confidence is low.
  • Complaint or negative sentiment appears.

What the user should do

  • Read reason and last message.
  • Use AI help for summary or draft reply.
  • Open conversation if context is needed.
  • Send a human response if required.
  • Resolve or snooze once handled.

Daily workflow

How users should operate the system.

The best user experience is simple: check urgent work first, then monitor batch health, then improve the agent.

Daily operator

  • Open Human Review.
  • Handle critical and high numbers.
  • Use AI help to draft replies.
  • Open Chats for full context.
  • Resolve after action.

Daily manager

  • Open AI CRM.
  • Check action required by batch.
  • Review buyer replies and cadence.
  • Open batch detail if needed.
  • Adjust plan or agent.

Weekly admin

  • Check WhatsApp credentials.
  • Review failed sends.
  • Improve knowledge base.
  • Test agent with new questions.
  • Clean duplicate batch ownership.

Quick troubleshooting order

If AI does not reply: check credentials, webhook admin mapping, batch AI enabled, duplicate phone ownership, agent enabled, handoff state, LLM provider error, and send error.

Best practices

How to get reliable AI sales replies.

The system works best when setup is clean and users treat the AI as a sales assistant with clear information, not as a magic box.

Do this

  • Use clear product and pricing knowledge.
  • Keep replies short and WhatsApp-friendly.
  • Test agent before enabling live follow-ups.
  • Use confirmation dialogs for manual sends.
  • Review Human Review every day.

Avoid this

  • Do not enable AI on messy duplicate batches blindly.
  • Do not leave old credentials active.
  • Do not use vague knowledge for pricing answers.
  • Do not ignore failed sends.
  • Do not let stale handoff block active batch AI.

Recommended first rollout

Start with one small batch, one configured agent, one clean knowledge set, and two verified WhatsApp test numbers. Send a template, reply from the phone, confirm AI response, then check Chats, AI Conversations, Human Review, and AI CRM.
